# 0070 Documentation-Reality Alignment Process

## Status

Accepted

## Context

Phase 3 found claims that were true in spirit but incomplete in user pathways.

## Decision

README claims must map to tests or be edited into limitations. Phase postmortems record supported and unsupported paths. Optional advanced modules are described as optional and gated.

## Consequences

Users get honest expectations and maintainers get a checklist for future releases.

## Alternatives Considered

- Leave README aspirational: rejected because documentation drift is a user-facing bug.
